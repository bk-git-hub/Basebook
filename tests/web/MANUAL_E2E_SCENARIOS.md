# Manual E2E Scenarios

로컬에서 `npm run dev`를 실행한 뒤 사람이 직접 확인하는 시나리오입니다.

## 1. 홈

1. `http://localhost:3000`을 연다.
2. Basebook 로고와 `서비스 알아보기` 링크를 확인한다.
3. 시즌 요약 카드와 최근 일지 목록을 확인한다.
4. `새 일지 남기기`와 `시즌 기록 전체 보기` 링크가 동작하는지 확인한다.

## 2. 소개 페이지

1. `/about`으로 이동한다.
2. 서비스가 경기 기록과 시즌 회고 중심으로 설명되는지 확인한다.
3. `새 기록 시작하기` 링크가 `/entries/new`로 이동하는지 확인한다.

## 3. 시즌 기록

1. `/season`으로 이동한다.
2. 시즌 요약, 직관 승률, 전체 기록 목록을 확인한다.
3. 기록 카드를 눌러 상세 화면으로 이동한다.

## 4. 새 기록 작성

1. `/entries/new`로 이동한다.
2. 날짜, 응원 팀, 상대 팀, 결과, 관람 형태, 한 줄 감상을 입력한다.
3. 직관일 때 경기장과 좌석 입력이 보이는지 확인한다.
4. TV, 모바일, 기타 관람으로 바꾸면 경기장과 좌석 입력이 숨겨지는지 확인한다.
5. 사진을 업로드하고 저장한다.
6. 저장 후 상세 화면으로 이동하는지 확인한다.

## 5. 기록 상세

1. `/entries/[id]`에서 경기 정보, 점수, 사진, 감상이 보이는지 확인한다.
2. `이 기록 수정하기` 링크가 수정 화면으로 이동하는지 확인한다.
3. `원본 열기` 링크가 업로드 이미지로 이동하는지 확인한다.

## 6. 기록 수정

1. `/entries/[id]/edit`으로 이동한다.
2. 기존 값이 폼에 채워져 있는지 확인한다.
3. 한 줄 감상이나 메모를 수정하고 저장한다.
4. 상세 화면에 수정 내용이 반영되는지 확인한다.

## 7. 기록 삭제

1. 상세 화면에서 `이 기록 삭제하기`를 누른다.
2. 삭제 후 `/season?entryDeleted=success`로 이동하는지 확인한다.
3. 삭제된 기록이 목록에서 사라졌는지 확인한다.
