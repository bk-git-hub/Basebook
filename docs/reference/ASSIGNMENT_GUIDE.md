## 1. 개발 과제

### 과제 목표

1인 개발자(파트너)라고 가정하고, `Book Print API`를 활용해 실제 서비스 가능한 웹 애플리케이션을 만든다.

핵심 목표:
- 최종 사용자(고객)가 있는 서비스를 구상할 것
- Book Print API를 연동할 것
- "콘텐츠를 책으로 만드는" 경험을 제공할 것

### 예시 아이디어

- 육아 일기 포토북
- 유치원 알림장 책
- AI 동화책
- 여행 포토북
- 1인 출판 플랫폼
- 반려동물 성장 앨범

위 예시에 얽매일 필요는 없고, 자신만의 아이디어도 가능하다.

### 진행 순서

`SDK 준비 및 저장소 생성 -> API 가입 및 Sandbox Key 발급 -> 웹앱 개발 -> GitHub 제출`

### Step 1. SDK 준비 및 저장소 생성

아래 두 SDK 중 하나를 선택해 GitHub 저장소에서 웹앱을 구축한다.

| 언어 | 저장소 |
|---|---|
| Python | `https://github.com/sweet-book/bookprintapi-python-sdk` |
| Node.js | `https://github.com/sweet-book/bookprintapi-nodejs-sdk` |

메모:
- SDK를 사용해도 되고 API를 직접 호출해도 됨
- SDK 언어와 백엔드 언어는 일치하지 않아도 됨
- 예: Python SDK 설치 + Node.js 백엔드 구현 가능

### Step 2. API 플랫폼 가입 및 Sandbox Key 발급

1. `https://api.sweetbook.com` 가입
2. 파트너 포털에서 Sandbox API Key 발급
3. Sandbox 환경에서 테스트 충전금 무료 충전 가능

메모:
- Sandbox는 테스트 전용 환경
- 실제 인쇄/배송은 이루어지지 않음

### Step 3. 개발

- 본인 GitHub 저장소에 웹 애플리케이션 코드를 추가
- 프론트엔드 + 백엔드를 하나의 저장소로 구성
- 주제 자유
- 기술 스택 자유

### Step 4. 제출

GitHub 저장소 URL을 서술형 문항이 있는 구글폼에 입력해 제출한다.
