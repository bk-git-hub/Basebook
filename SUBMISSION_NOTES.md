# Submission Notes

## 2026-04-15

- 후보 문항: Q4. AI 도구를 쓰면서 겪은 실패 또는 문제, 그리고 대응 방식
- 맥락: 저장소 규칙상 모든 의미 있는 작업은 `plan -> technical meeting with user -> implementation -> test -> review -> user verification` 순서를 따라야 했고, 구현 전에는 사용자 승인 대기가 필수였다.
- 문제: `/about` 화면 톤 정리 작업에서 AI가 사용자의 지시를 곧바로 구현 승인으로 오해했다. 짧게 방향만 보고한 뒤, 승인 대기 없이 실제 수정과 커밋까지 진행했다.
- 발견 방식: 사용자가 "왜 수정 계획 보고 없이 바로 수정에 들어갔지"라고 지적하면서 승인 스킵이 드러났다. 사용자는 이것을 AI 협업에서 반복적으로 겪는 주요 시행착오라고 설명했다.
- 원인 정리: 기본 자율 실행 성향이 저장소의 로컬 협업 규칙보다 앞서 적용되었고, "보고"를 단순 진행 안내로 잘못 해석했다. 승인 게이트를 하드 스톱으로 다루지 못했다.
- 대응: AI가 즉시 판단 오류를 인정했고, 앞으로는 작업 시작 전 계획 보고 후 반드시 사용자 승인 표현(예: `진행해`, `승인`, `A안으로 가자`)을 받은 뒤에만 파일 수정에 들어가기로 원칙을 재확인했다.
- 교훈: AI는 일반적인 "바로 실행" 성향보다 저장소별 협업 규칙을 우선해야 한다. 특히 승인 기반 워크플로우에서는 "보고"와 "실행 허가"를 분리해서 다뤄야 한다.
